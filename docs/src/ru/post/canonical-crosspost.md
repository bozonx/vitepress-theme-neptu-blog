**Каноническая ссылка (canonical)** указывает поисковым системам, какой URL является
первоисточником и основной версией страницы, предотвращая санкции за дублирование контента.
Этот пост объявляет *внешний* canonical — если изучить `<head>`, вы увидите, что он указывает на
`example.com`, а не на этот сайт.

## Поведение по умолчанию

При `autoCanonical: true` (по умолчанию) каждая страница автоматически получает каноническую
ссылку на саму себя. Для обычных постов вам ничего делать не нужно.

## Кросспостинг: внешний canonical

Когда вы перепубликуете статью, которая первоначально выходила в другом месте, укажите каноническую ссылку
на оригинальную публикацию, чтобы сохранить SEO-рейтинг первоисточника.

### Как это сделать

```yaml
canonical: "https://example.com/ru/articles/original-post"
```

Используйте этот параметр при синдикации собственного контента, импорте со старого домена
или перепубликации по разрешению.

## Принудительная каноническая ссылка на себя

Если вы отключили `autoCanonical` глобально, но хотите, чтобы *эта* страница ссылалась
на саму себя:

```yaml
canonical: "self"   # или сокращение: "s"
```

## Отключение канонической ссылки для одной страницы

```yaml
seo:
  canonical: false
```

## Глобальные настройки

```yaml
# src/site.yaml
themeConfig:
  seo:
    canonical: true
    autoCanonical: true
```

Сведения о hreflang (многоязычном аналоге canonical) см. в
[Локализация и hreflang](i18n-hreflang).
