Карточки постов в списках (например, на страницах **Свежие посты**, **Теги** и **Авторы**)
отображают краткое превью. По умолчанию тема генерирует превью из начала
текста поста, но его можно переопределить — что и сделано в этом посте. Откройте
[список свежих постов](../recent/1) и найдите этот пост, чтобы сравнить текст его карточки
с абзацем, который вы читаете сейчас.

## Три способа управления превью

**1. Автоматический (по умолчанию).** Ничего не делайте — тема сделает выжимку из текста статьи
длиной до `maxPreviewLength` символов.

**2. Использование описания.** Установите `descrAsPreview: true`, чтобы использовать поле `description`
из фронтматера как текст карточки (так сделано в большинстве демо-постов).

**3. Явный текст.** Задайте `previewText`, чтобы вручную написать текст карточки. Используйте
`descrAsPreview` **или** `previewText`, но не оба вместе.

### Как это сделать

```yaml
# Этот пост использует явный текст превью:
previewText: "Текст этой карточки задан явно через previewText…"

# Или альтернативный вариант:
# descrAsPreview: true
```

## Длина превью

Ограничение длины анонса задаётся в коде, поскольку загрузчик данных считывает его при сборке:

```ts
// .vitepress/config.ts
export const postList = { maxPreviewLength: 300 }
```

Отображение элементов карточки (дата, теги, миниатюра, автор) настраивается в YAML:

```yaml
# src/site.yaml
themeConfig:
  postList:
    showDate: true
    showTags: true
    showThumbnail: true
    showPreview: true
    showAuthor: true
```

## Исключение поста из поиска

Поиск работает на основе Pagefind. Чтобы не включать пост в поисковый индекс, укажите:

```yaml
searchIncluded: false
```

Пост по-прежнему будет отображаться на сайте и попадать в списки; он просто не будет появляться
в результатах поиска. Подробнее о настройке поиска см. в разделе [Ленты, поиск и SEO-переключатели](../page/seo-feeds-search).
